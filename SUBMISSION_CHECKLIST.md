## Common Rejection Reasons to Avoid

### iOS
- [ ] No placeholder content or incomplete features
- [ ] No broken links in app or metadata
- [ ] No misleading screenshots
- [ ] No excessive ads or intrusive monetization
- [ ] Proper handling of permissions
- [ ] Background modes justified (if used)
- [ ] Sign In with Apple implemented (if using social login)

### Android
- [ ] No deceptive behavior
- [ ] No excessive permissions requested
- [ ] Proper handling of background location (if used)
- [ ] No malware or harmful code
- [ ] Accurate metadata and descriptions
- [ ] Compliance with Families Policy (if targeting children)
